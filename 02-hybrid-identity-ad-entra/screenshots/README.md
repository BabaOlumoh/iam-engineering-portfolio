## Screenshot Guidance

Avoid exposing secrets, tenant IDs, recovery codes, private IPs if not needed, and personal user data. Use lab accounts only.

For each screenshot, try to capture:

- What was configured.
- Why it matters.
- The test result or success state.

Example evidence description:

```text
Screenshot: 06-entra-synced-user.png
Description: Shows a test user synchronized from on-premises AD to Microsoft Entra ID with On-premises sync enabled set to Yes and UPN using 0gkareemu.live.
```
